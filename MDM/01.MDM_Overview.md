# 📂 케이스 스터디: 이기종 데이터 통합 및 마스터 데이터 관리 (MDM)
> **주제:** BI-Matrix를 활용한 10여 개 계열사 대상 비용 효율적 마스터 데이터 관리 시스템 구축

## 1. 개요 (Executive Summary)
본 프로젝트는 기업 환경에서 흔히 발생하는 **데이터 파편화(Data Fragmentation)** 문제를 해결하기 위해 진행되었습니다. 10개 이상의 계열사가 서로 다른 ERP 시스템을 사용함에 따라 인사 데이터의 기준이 불일치하는 문제가 있었으나, 이를 해결하기 위해 고가의 MDM 솔루션 대신 기존 인프라인 **BI-Matrix**를 활용하여 자체 매핑 시스템을 설계했습니다. 이를 통해 그룹 표준 코드와 각 계열사의 로컬 코드를 동기화하여 전사 통합 분석이 가능해졌습니다.

---

## 2. 문제 정의 (Problem Statement)
* **데이터 실로(Data Silos):** 계열사별 서로 다른 코드 체계로 인해 그룹 전체의 인력 현황을 집계하는 데 막대한 시간 소요.
* **비용 장벽:** 상용 MDM 솔루션 도입에는 높은 초기 비용과 유지보수비가 발생하여 재무적 부담이 큼.
* **분석의 한계:** 수작업으로 데이터를 정제하고 매핑하는 과정에서 휴먼 에러가 발생하며 실시간 분석이 불가능함.

---

## 3. 구현 내용: 통합 인사 마스터 코드 인터페이스
이미 도입된 **BI-Matrix** 플랫폼의 스크립트 기능을 활용하여, 데이터 조회 및 조작(CRUD)이 가능한 동적 관리 화면을 개발했습니다.

### 🌟 주요 기능 (Key Features)
* **듀얼 그리드 동기화 (Recursive Mapping):** 단일 테이블 내에서 셀프 참조 구조를 가집니다. **Top Level Grid**에서 카테고리(예: 직급, 학력)를 선택하면 하단의 **Detail Grid**에 해당되는 상세 매핑 정보가 즉시 필터링되어 표시됩니다.
* **동적 메타데이터 헤더 (Dynamic Metadata Header):** 코드 종류에 따라 관리 항목이 다르므로, 선택된 카테고리에 맞춰 하단 그리드의 컬럼 명칭(`VALUE1`~`VALUE10`)이 실시간으로 변경되는 스크립트를 구현했습니다.
* **직접적 데이터 제어:** 사용자가 화면에서 직접 **Search, Add, Save** 버튼을 통해 매핑 로직을 수정할 수 있어 데이터 관리의 자율성을 확보했습니다.

## 4. 데이터 아키텍처 (Technical Architecture)

### 데이터베이스 스키마: `Integrated_Master_Code`
확장성을 고려하여 설계된 테이블 구조로, 별도의 스키마 변경 없이도 다양한 계열사의 속성값을 수용할 수 있습니다.

| Column Name | Type | Nullable | Description |
| :--- | :--- | :--- | :--- |
| `SEQ` | BIGINT | No | 레코드 식별을 위한 기본키 (PK) |
| `CATEGORY_CODE` | VARCHAR(20) | No | 상위 카테고리 식별자 (예: `RankCl`, `GenderCl`) |
| `DETAIL_CODE` | VARCHAR(30) | No | 카테고리 내 상세 항목 코드 |
| `CATEGORY_NAME` | VARCHAR(20) | Yes | UI에 표시될 카테고리 명칭 |
| `DETAIL_NAME` | VARCHAR(100) | Yes | 표준화된 항목 명칭 |
| `VALUE1 ~ VALUE10` | VARCHAR(Varies) | Yes | **매핑 필드**: 계열사별 로컬 코드 및 속성값 저장 |
| `SORT_ORDER` | INT | Yes | 화면 출력 순서 |
| `UPDATED_BY` | VARCHAR(10) | Yes | 수정자 ID (감사 추적용) |
| `UPDATED_AT` | VARCHAR(20) | Yes | 수정 일시 (감사 추적용) |

## 5. 엔지니어링 임팩트 및 비즈니스 가치

### 🚀 기술적 우수성: "설정 기반의 확장성 (Scalable Design)"
이 시스템은 단순한 관리 도구를 넘어, 데이터 파이프라인의 **메달리온 아키텍처(Medallion Architecture)** 중 Gold Layer의 설정 도구 역할을 수행합니다. 새로운 계열사가 추가되더라도 개발자의 개입 없이 화면에서 행을 추가하는 것만으로 데이터 통합이 가능합니다.

### 💰 전략적 비용 절감
상용 솔루션 대신 자체 구축을 통해 다음과 같은 성과를 거두었습니다.
* **솔루션 라이선스 비용 100% 절감**
* **리포트 생성 시간 90% 이상 단축** (수작업 엑셀 정제 방식 대비)
